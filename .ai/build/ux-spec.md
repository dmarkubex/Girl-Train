# UX Specification - 脊柱侧弯锻炼追踪器 PWA

## Pages & Components

### 首页 (Home)
**Prototype file:** prototype/home.html
**User goal:** 快速开始今日锻炼，查看锻炼坚持情况
**Entry points:** 应用启动、返回导航
**Primary action:** 点击"开始锻炼"按钮
**Secondary actions:** 查看历史记录、编辑锻炼配置、查看今日状态

### 锻炼执行页 (Workout)
**Prototype file:** prototype/workout.html
**User goal:** 跟随自动计时完成全部锻炼项目
**Entry points:** 首页点击"开始锻炼"
**Primary action:** 暂停/恢复计时
**Secondary actions:** 跳过当前组、提前结束锻炼
**Key features:** 全屏倒计时显示、进度条、阶段指示器

### 完成统计页 (Complete)
**Prototype file:** prototype/complete.html
**User goal:** 查看本次锻炼成果，记录感受
**Entry points:** 锻炼自然完成、提前结束
**Primary action:** 保存备注（可选）、返回首页
**Secondary actions:** 查看详细统计、对比历史数据

### 历史记录页 (History)
**Prototype file:** prototype/history.html
**User goal:** 追溯锻炼历史，查看趋势和坚持情况
**Entry points:** 首页导航入口
**Primary action:** 浏览日历和趋势图
**Secondary actions:** 点击日期查看详情、切换周/月视图

### 配置管理页 (Config)
**Prototype file:** prototype/config.html
**User goal:** 调整锻炼计划、管理数据
**Entry points:** 首页导航入口
**Primary action:** 保存配置修改
**Secondary actions:** 添加/删除项目、调整顺序、导出/导入数据

## Interaction Rules

### 表单提交
- 所有表单输入即时验证，错误提示显示在字段下方
- 保存操作显示 2 秒成功提示，然后返回或保持当前页面
- 无效输入阻止提交，聚焦到第一个错误字段

### 空状态
- **首页无历史记录:** 显示引导文案"开始你的第一次锻炼"
- **历史记录页无数据:** 显示空日历和"完成第一次锻炼后这里会显示趋势"
- **配置页无项目:** 显示"添加第一个锻炼项目"引导

### 确认对话框
以下操作需要二次确认：
- 提前结束锻炼（显示"已完成的进度将被保存"）
- 删除锻炼项目
- 导入数据覆盖配置
- 放弃恢复未完成的锻炼

### 导航
- 底部固定导航栏（首页、历史、配置）
- 锻炼执行页隐藏导航栏（全屏沉浸式）
- 页面切换使用淡入淡出动画（200ms）
- 硬件返回键提示保存未保存的更改

## Operation Hierarchy

### 首页 (Home)
- **PRIMARY (醒目 CTA):** 开始锻炼按钮 - 大号圆形渐变按钮，居中显示
- **SECONDARY:** 查看历史记录、编辑配置 - 小图标按钮，右上角或底部
- **TERTIARY:** 连续天数徽章、今日状态卡片 - 信息展示

### 锻炼执行页 (Workout)
- **PRIMARY:** 暂停/恢复按钮 - 大号圆形按钮，底部居中
- **SECONDARY:** 跳过按钮 - 中等按钮，底部右侧
- **DANGER:** 结束锻炼 - 红色按钮，底部左侧，需要确认

### 完成统计页 (Complete)
- **PRIMARY:** 返回首页按钮 - 大号按钮
- **SECONDARY:** 保存备注按钮 - 次要按钮
- **TERTIARY:** 查看详情 - 折叠/展开详细信息

### 历史记录页 (History)
- **PRIMARY:** 切换周/月视图 - 标签切换
- **SECONDARY:** 点击日期查看详情
- **TERTIARY:** 滚动浏览历史、导航到上月/下月

### 配置管理页 (Config)
- **PRIMARY:** 保存配置按钮 - 固定底部按钮
- **SECONDARY:** 添加项目、调整顺序（上/下箭头）
- **DANGER:** 删除项目、导出/导入数据

## Edge Cases

### 1. 跨夜锻炼场景
**场景:** 用户 23:45 开始锻炼，00:20 完成
**行为:** 记录归属开始日期（前一天），streak 正常计算
**显示:** 完成统计页显示"归属于 4月10日"小字提示

### 2. 锻炼中途意外退出
**场景:** 用户锻炼中误触返回键或浏览器崩溃
**行为:** 下次打开显示恢复对话框"检测到未完成的锻炼"
**选项:** 继续锻炼（恢复状态）/ 放弃（保存已完成部分）

### 3. 首次使用无配置
**场景:** 用户首次打开应用
**行为:** 自动加载默认配置（抱头展胸、上胸弯对抗）
**提示:** 显示"已为你准备默认锻炼计划，可在配置中调整"

### 4. 连续天数中断
**场景:** 用户连续锻炼 7 天后第 8 天未锻炼
**行为:** 首页显示"连续锻炼 0 天"，历史最长显示"最长连续 7 天"
**视觉:** 连续天数徽章变为灰色，鼓励文案"重新开始，你能做到！"

### 5. 导入数据冲突
**场景:** 用户导入包含相同 sessionId 的数据
**行为:** sessions 跳过已存在记录，config 提示"本地配置将被覆盖，是否继续？"
**选项:** 覆盖配置 / 取消导入

### 6. 音频播放失败
**场景:** iOS Safari 音频被系统静音
**行为:** 同时使用振动反馈和屏幕闪烁提示
**降级:** 纯视觉倒计时（文字变色 + 进度条动画）

### 7. 计时器后台运行
**场景:** 用户锁屏或切换应用
**行为:** 使用 performance.now() 计算实际经过时间，恢复时校准
**显示:** 返回时显示"离开期间计时继续"短暂提示

### 8. 空备注保存
**场景:** 完成统计页用户未输入备注直接返回
**行为:** 自动保存，备注字段为空字符串
**无干扰:** 不显示任何提示，直接返回首页

### 9. 配置参数无效
**场景:** 用户输入负数时长或 0 组数
**行为:** 实时验证，字段下方红色提示"时长必须大于 0"
**阻止:** 保存按钮保持禁用状态

### 10. 日历无数据月份
**场景:** 用户查看无锻炼记录的月份
**行为:** 日历显示空白日期（灰色），趋势图显示平直线（0%）
**提示:** "本月无锻炼记录，开始你的第一次锻炼吧！"

## Design Decisions

### 视觉风格
- **主色调:** 温暖的橙色系（#FF8C42）传达积极能量，搭配深绿色（#2E7D32）表示完成状态
- **字体:** 思源黑体（Noto Sans SC）- 清晰易读的中文字体，数字使用等宽数字字体
- **布局:** 移动端优先 375px 基准，大按钮（最小 44x44px touch target）、充足留白
- **动画:** 微妙的淡入淡出（200ms）、进度条平滑过渡（300ms）、按钮按下反馈（100ms）

### 信息层级
- **首页:** 一目了然 - 大开始按钮 + 简洁状态卡片
- **锻炼页:** 极简专注 - 巨大倒计时数字 + 清晰阶段标签
- **统计页:** 数据可视化 - 环形图吸引注意力，柱状图提供细节
- **历史页:** 时间探索 - 日历宏观 + 趋势图微观
- **配置页:** 功能性表单 - 结构化列表，操作按钮就近

### 无障碍考虑
- 所有交互元素支持键盘导航
- 颜色对比度符合 WCAG AA 标准
- 重要操作提供语音/振动双重反馈
- 暗色模式支持（系统自动切换）

### 性能优化
- 静态资源预加载
- 图表按需加载
- 动画使用 CSS transform（GPU 加速）
- localStorage 快照异步写入（不阻塞 UI）
